好的 Agent 執行不是只靠一句指令，而是來自清楚的操作框架: 目標明確、範圍可控、工具合適，且下一步足夠清楚，能安全往前推進。

Workflow

Planning / Delegation / Checks

這份文件也重視執行紀律: 讓 Agent 先拆解工作，再由工具接觸環境， 最後用審查與檢查流程收斂風險與品質。

01 / Snapshot

先快速掌握這份文件怎麼看待 Agent 工作。

如果你想先抓大圖，可以從這裡開始。這頁的目標是替團隊建立一套討論 Agent 工作流的共同語言， 從規劃、工具執行，一路連到委派與審查方式。

Core Frame

Goal / Context / Action

強的 Agent 通常來自乾淨的任務 framing: 目標明確、背景相關、下一步可執行， 這樣它就不用靠猜測補洞。

Execution

工具落地執行

在真實工作流裡，Agent 不該只停在聊天。工具讓它能搜尋、讀檔、改檔、跑檢查， 並把假設拿去對照真實環境。

Tooling

Review / Handoff / Repeatability

這份文件把審查與交接當成 Agent 工作的一部分，讓結果能被檢查、能被接手， 也能變成可重複的方法，而不是只靠一次幸運產出。

02 / Reading Path

建議的閱讀順序，幫你更快吸收重點。

這份導讀最適合從任務 framing 讀到工具，再讀到審查與交接。 這樣比較容易把抽象的 Agent 概念轉成可靠的運作方式。

Suggested Flow

閱讀路徑

  1. 先看任務如何被定義、切分，以及範圍如何被收斂。
  2. 再看 Agent 如何使用工具、背景資訊與委派機制往前推進。
  3. 最後把審查、交接與可重複流程包起來看成完整系統。
Use Cases

適用情境

  • 替剛開始接觸 AI Agent 的成員建立 onboarding 材料。
  • 設計工程、研究或內容工作可重複使用的 Agent 流程。
  • 檢查委派、工具使用或驗證流程為什麼卡住。
  • 把一次性的實驗整理成團隊可重用的方法。
